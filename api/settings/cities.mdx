## List Cities

<api method="GET" endpoint="/cities" auth="public">
  Get a paginated list of cities
</api>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number (minimum: 1)
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Items per page (minimum: 1, maximum: 100)
</ParamField>

<ParamField query="q" type="string">
  Free text search by name (case-insensitive)
</ParamField>

<ParamField query="name" type="string">
  Filter by exact name
</ParamField>

<ParamField query="countryCode" type="string">
  Filter by ISO-3166-1 alpha-2 country code (e.g., MX, CO)
</ParamField>

<ParamField query="timezone" type="string">
  Filter by IANA timezone (e.g., America/Mexico_City)
</ParamField>

<ParamField query="active" type="boolean">
  Filter by active status
</ParamField>

### Response

Returns a paginated list of cities with metadata.

## Create City

<api method="POST" endpoint="/cities" auth="public">
  Create a new city
</api>

### Body Parameters

<ParamField body="name" type="string" required>
  City name (max 120 characters)
</ParamField>

<ParamField body="countryCode" type="string" required>
  ISO-3166-1 alpha-2 country code (e.g., MX, CO)
</ParamField>

<ParamField body="timezone" type="string" required>
  Valid IANA timezone (e.g., America/Mexico_City)
</ParamField>

<ParamField body="geom" type="object">
  GeoJSON MultiPolygon representing the city boundaries
</ParamField>

<ParamField body="active" type="boolean" default="true">
  Whether the city is active
</ParamField>

### Response

<ResponseField name="status" type="string">
  Response status
</ResponseField>

<ResponseField name="data" type="object">
  The created city object
</ResponseField>

### Error Responses

- **409 Conflict**: City already exists (name + countryCode combination)
- **400 Bad Request**: Validation error

## Update City

  Update a city (partial update)
</api>

<ParamField path="id" type="string" required>

  City name (max 120 characters)
</ParamField>

</ParamField>

</ParamField>

## Resolve City by Point

<api method="GET" endpoint="/cities/resolve/by-point" auth="public">
  Find the city containing a geographic point
</api>

### Query Parameters

<ParamField query="lat" type="number" required>
  Latitude coordinate
</ParamField>

<ParamField query="lng" type="number" required>
  Longitude coordinate
</ParamField>

### Response

Returns the city containing the point, or null if no city is found.

### Example Request

```bash
curl "https://api.rodando.com/cities/resolve/by-point?lat=19.4326&lng=-99.1332"
```

### Example Response

```json
{
  "success": true,
  "message": "City resolved",
  "data": {
    "id": "a1b2c3d4-5678-90ab-cdef-1234567890ab",
    "name": "Ciudad de México",
    "countryCode": "MX",
    "timezone": "America/Mexico_City",
    "active": true,
    "createdAt": "2024-01-15T10:30:00Z",
    "updatedAt": "2024-01-15T10:30:00Z"
  }
}
```

If no city contains the point:

```json
{
  "success": true,
  "message": "No city found",
  "data": null
}
```

### Use Cases

This endpoint is essential for:

- **Trip pricing** - Determine which city's pricing policies apply to a trip
- **Service availability** - Check if Rodando operates in the requested location
- **Zone resolution** - First step before resolving specific zones within a city

<Note>
  This endpoint uses PostGIS spatial queries to find the city whose geometry contains the provided point. Only active cities with defined geometries are considered.
</Note>
