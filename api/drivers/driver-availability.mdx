## Create Driver Availability

<api method="POST" path="/driver-availability">
  Creates a new driver availability record
</api>

### Body Parameters

<ParamField body="driverId" type="string" required>
  UUID of the driver
</ParamField>

<ParamField body="isOnline" type="boolean">
  Whether the driver is currently online. Defaults to `false`
</ParamField>

<ParamField body="isAvailableForTrips" type="boolean">
  Whether the driver is available to accept trips. Defaults to `false`
</ParamField>

<ParamField body="lastLocation" type="object">
  Driver's last known location
  <Expandable title="properties">
    <ParamField body="lat" type="number" required>
      Latitude coordinate (-90 to 90)
    </ParamField>
    <ParamField body="lng" type="number" required>
      Longitude coordinate (-180 to 180)
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="lastLocationTimestamp" type="string">
  Timestamp of the last location update (ISO 8601)
</ParamField>

<ParamField body="currentTripId" type="string">
  UUID of the current active trip, or `null` if no active trip
</ParamField>

<ParamField body="currentVehicleId" type="string">
  UUID of the vehicle currently being used, or `null` if none
</ParamField>

<ParamField body="lastOnlineTimestamp" type="string">
  Timestamp when the driver last went online (ISO 8601)
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Response message
</ResponseField>

<ResponseField name="data" type="object">
  Created driver availability record
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Availability record UUID
    </ResponseField>
    <ResponseField name="driverId" type="string">
      Driver UUID
    </ResponseField>
    <ResponseField name="isOnline" type="boolean">
      Online status
    </ResponseField>
    <ResponseField name="isAvailableForTrips" type="boolean">
      Trip availability status
    </ResponseField>
    <ResponseField name="lastLocation" type="object">
      Last known location coordinates
    </ResponseField>
    <ResponseField name="currentTripId" type="string">
      Current trip UUID
    </ResponseField>
    <ResponseField name="currentVehicleId" type="string">
      Current vehicle UUID
    </ResponseField>
    <ResponseField name="updatedAt" type="string">
      Last update timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

## Ping Location

<api method="POST" path="/driver-availability/ping-location">
  Updates driver location and heartbeat (authenticated, rate limited: 8 pings per 10 seconds)
</api>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Body Parameters

<ParamField body="lat" type="number">
  Current latitude (if provided, triggers location update)
</ParamField>

<ParamField body="lng" type="number">
  Current longitude (if provided, triggers location update)
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Response message
</ResponseField>

<ResponseField name="data" type="object">
  Updated availability record
</ResponseField>

<Note>
  This endpoint is rate-limited to prevent excessive updates. If lat/lng are provided, the location is updated only if the driver has moved beyond a minimum threshold. Without coordinates, it acts as a heartbeat ping.
</Note>
